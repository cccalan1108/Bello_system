### 1.6 資料庫設計
- **PostgreSQL 資料庫架構**
  - 資料庫名稱: bello
  - 資料庫連線設定完成 (host: 127.0.0.1, port: 5432)
  - UTF-8 編碼設定完成

- **9 張資料表設計**
  - **USER**: 使用者基本資料表
    - 主鍵: User_id (BIGINT)
    - 唯一約束: Account, Email
    - 欄位: Account, User_name, User_nickname, Password, Nationality, City, Phone, Email, Sex, Birthday, Register_time
  
  - **USER_DETAIL**: 使用者詳細資料表
    - 複合主鍵: (User_id, Self_introduction)
    - 外鍵: User_id → USER(User_id) ON DELETE CASCADE
    - CHECK 約束: Star_sign, Mbti, Blood_type, Religion, Married, Sns
    - 欄位: Star_sign, Mbti, Blood_type, Religion, University, Married, Sns, Self_introduction, Interest, Find_meeting_type
  
  - **USER_ROLE**: 使用者角色表
    - 複合主鍵: (User_id, Role)
    - 外鍵: User_id → USER(User_id) ON DELETE CASCADE
    - CHECK 約束: Role IN ('User', 'Admin')
  
  - **MEETING**: 聚會資料表
    - 主鍵: Meeting_id (BIGINT)
    - 外鍵: Holder_id → USER(User_id) ON DELETE CASCADE
    - CHECK 約束: Content IN ('午餐', '咖啡/下午茶', '晚餐', '喝酒', '語言交換')
    - CHECK 約束: Status IN ('Ongoing', 'Finished', 'Canceled')
    - 欄位: Holder_id, Content, Event_date, Start_time, End_time, Event_city, Event_place, Status, Num_participant, Max_num_participant
  
  - **MEETING_LANGUAGE**: 聚會語言表
    - 複合主鍵: (Meeting_id, Language)
    - 外鍵: Meeting_id → MEETING(Meeting_id) ON DELETE CASCADE
    - CHECK 約束: Language (15 種語言選項)
  
  - **PARTICIPATION**: 參與記錄表
    - 複合主鍵: (User_id, Meeting_id)
    - 外鍵: User_id → USER(User_id) ON DELETE CASCADE
    - 外鍵: Meeting_id → MEETING(Meeting_id) ON DELETE CASCADE
    - 欄位: Join_time (TIMESTAMP)
  
  - **SNS_DETAIL**: 社群媒體帳號表
    - 複合主鍵: (User_id, Sns_type, Sns_id)
    - 外鍵: User_id → USER(User_id) ON DELETE CASCADE
    - CHECK 約束: Sns_type (10 種平台選項)
  
  - **CHATTING_ROOM**: 聚會聊天室表
    - 複合主鍵: (Meeting_id, Sender_id, Sending_time)
    - 外鍵: Meeting_id → MEETING(Meeting_id) ON DELETE CASCADE
    - 外鍵: Sender_id → USER(User_id) ON DELETE CASCADE
    - 欄位: Content (VARCHAR(200))
  
  - **PRIVATE_MESSAGE**: 私人訊息表
    - 複合主鍵: (Sender_id, Receiver_id, Sending_time)
    - 外鍵: Sender_id → USER(User_id) ON DELETE CASCADE
    - 外鍵: Receiver_id → USER(User_id) ON DELETE CASCADE
    - 欄位: Content (VARCHAR(200))

- **資料完整性設定**
  - 外鍵約束完整實作 (ON DELETE CASCADE, ON UPDATE CASCADE)
  - CHECK 約束實作 (確保資料值符合規範)
  - UNIQUE 約束實作 (Account, Email)
  - NOT NULL 約束實作 (必要欄位)

- **模擬資料**
  - 10,000 筆使用者模擬資料匯入完成 (users.csv)
  - 資料匯入機制實作 (\copy 指令)
